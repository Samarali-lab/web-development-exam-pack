Section B — Short Answer (Key points)

  1. Specificity: inline > id > class/attr/pseudo-class > element/pseudo-element. Example: #a overrides .a overrides p.
  2. Float wraps content; positioning places element by offsets (absolute/fixed etc.).
  3. Absolute: positioned ancestor; Fixed: viewport; fixed stays while scrolling.
  4. Use table with header row (th) and data rows (td), optional caption.
  5. Validation attrs: required, type, minlength, maxlength, pattern, min, max, step.
  6. Function: function f(){}; call: f(); (or function expression).
  7. Class toggle: el.classList.toggle("active") inside click handler.
  8. Traversal: parentNode, children, nextElementSibling, previousElementSibling.
  9. Indeterminate = partial selection state (JS: master.indeterminate=true).
  10. Debug: breakpoints, inspect elements, console logs, watch variables.

Section C — Long Answer (Outlines)

  1. Registration: layout with labels/inputs; CSS; JS check required/pattern; show errors; prevent submit.
  2. DOM ops: createElement, createTextNode; appendChild; removeChild/remove; replaceChild; cloneNode.
  3. Events: click/mouseover/keydown; addEventListener; bubbling; stopPropagation when needed.
  4. Music player UI: header, controls, progress, playlist list; semantic buttons and input range.

Section D — Practical (Logic)

  1. Clock: setInterval every 1000ms; new Date; update DOM text/rotate hands with CSS transforms.
  2. Navbar dropdown: hover/focus show submenu; media query for mobile stacking.
  3. Selection: master checkbox set all; update master checked/indeterminate based on items.
