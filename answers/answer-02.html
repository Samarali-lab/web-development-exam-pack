Section B — Short Answer (Key points)

  1. Hotspot: <img usemap="#m"> + <map name="m"><area shape coords href>...</map>.
  2. Pseudo-class: state (:hover, :focus); pseudo-element: part (::before, ::first-line).
  3. Margin = outside space; padding = inside space between content and border.
  4. Clear floats: clearfix (pseudo-element) OR parent overflow:auto/hidden OR clear:both element.
  5. GET: data in URL, limited, cacheable. POST: data in body, safer for large/sensitive.
  6. Use required, correct type, minlength, pattern, maxlength.
  7. Element node = tags; text node = text inside element.
  8. JS style: el.style.color="red"; el.style.backgroundColor="...".
  9. el.getAttribute("href"), el.setAttribute("title","x").
  10. window.onload runs after page loads; safe to access DOM.

Section C — Long Answer (Outlines)

  1. Navbar: ul/li links; CSS hover; flex layout; media query to stack; active link styling.
  2. Registration: semantic form groups; labels; required validations; hints/errors; CSS layout; JS final validation.
  3. DnD bin: dragstart set data, dragover preventDefault, drop read data and remove element.
  4. Arrays + if/switch + loops (for/while); examples: iterate array; condition checks; form validation.

Section D — Practical (Logic)

  1. Use draggable="true"; events: dragstart/dragover/drop; on drop delete node.
  2. Questionnaire: radio/checkbox/select; required checks on submit; show message if not selected.
  3. Olympic rings: 5 circles using div with border-radius:50%; colored borders; position with relative/absolute or flex.
