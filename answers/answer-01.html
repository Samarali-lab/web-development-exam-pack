Section B — Short Answer (Key points)

  1. Block: new line, full width (div, p). Inline: same line, needed width (span, a).
  2. Text: <a href="page.html">Open</a>; Image: <a href="page.html"><img src="a.jpg" alt="A"></a>; Email: <a href="mailto:x@y.com">Mail</a>.
  3. Id: #x unique. Class: .c reusable. Element: p targets all p.
  4. DOM = Document Object Model; HTML becomes a node tree (document→html→head/body→elements/text).
  5. <table><caption>...</caption><tr><th>...</th></tr><tr><td>...</td></tr></table>.
  6. :focus focused element; :active while clicking; :visited visited link.
  7. Float = element floats left/right and text wraps. Problems: parent height collapse; overlap/clearing issues.
  8. action=submit URL, method=GET/POST, name=field key, required=must fill.
  9. == type conversion; === strict (type+value).
  10. alert message; confirm OK/Cancel boolean; prompt input string/null.

Section C — Long Answer (Exam-style outlines)

  1. Invitation letter: use background (body), container div/card, headings, paragraphs, list; CSS for font, color, spacing, alignment.
  2. Positioning: static default; relative offsets but keeps space; absolute removed from flow relative to positioned ancestor; fixed relative to viewport.
  3. Controls: text/password inputs; radio grouped by same name; checkbox multi; select+option; attributes: value, checked, selected, disabled, placeholder.
  4. DOM: select (getElementById/querySelector), read/write attributes (getAttribute/setAttribute), change text (innerText), change style (element.style).

Section D — Practical (Exam-style logic)

  1. Navbar: flex/inline list; hover styles; media query to set display:block on links and stack items.
  2. Select all: master checkbox toggles all; update master checked & indeterminate based on checked count.
  3. Create li: document.createElement("li"), set text, ul.appendChild(li) on button click.
